What this tool does

      

        Underwater footage loses red first, then orange, then yellow with depth — leaving everything
        green-cyan. color-correct runs a scene-adaptive enhancement chain (auto white-balance,
        Ancuti red-channel recovery, per-channel tone LUT, temporal smoothing) directly in your browser.
      

      

        Decode and encode use your device's hardware video chip via WebCodecs. Color processing runs
        as a WebGL2 fragment shader on the GPU. Output is wrapped into a standard .mp4 with
        the original audio passed through unchanged. Nothing is uploaded.
      


      
Pipeline

      
input .mp4
   │
   ├─ mp4box.js        demux container
   ├─ VideoDecoder     hardware decode → VideoFrame
   ├─ WebGL2 shader    Ancuti red recovery + per-channel LUT + strength
   ├─ VideoEncoder     hardware H.264 encode (25 Mbps VBR, bf=0)
   └─ mp4-muxer        wrap into .mp4 (audio passed through)
   │
   ▼  download blob


      
Output specs

      

        H.264 High @ Level 5.2 · 8-bit yuv420p · BT.709 TV range · no B-frames · ~25 Mbps VBR ·
        original AAC bit-exact passthrough · faststart for streaming playback.
      


      
Browser support

      

        Chrome ≥ 94 (Android, desktop) · Edge ≥ 94 · Samsung Internet ≥ 26 · Safari ≥ 17.4 (iOS, macOS).
        Firefox is rolling out WebCodecs gradually — run the Probe tab to confirm.
